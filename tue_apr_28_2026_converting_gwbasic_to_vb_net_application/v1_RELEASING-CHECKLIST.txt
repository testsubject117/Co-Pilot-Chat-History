Date: ____________

VERSIONING POLICY (SemVer-ish)
- Patch (bugfix):        1.0.0 -> 1.0.1      (AssemblyVersion stays the same)
- Minor (new features):  1.0.0 -> 1.1.0      (AssemblyVersion stays the same)
- Major (breaking):      1.x   -> 2.0.0      (AssemblyVersion -> 2.0.0.0)

In AMiOffice.vbproj:
- Version = X.Y.Z
- AssemblyVersion = A.0.0.0 (A changes only on major)
- FileVersion = $(Version).$(BuildNumber) (auto changes on Publish)
- InformationalVersion = $(Version) (Build $(BuildNumber))

------------------------------------------------------------
1) PRE-FLIGHT
[ ] Confirm app runs locally (Debug/Release build succeeds)
[ ] Confirm data files present on dev machine OR test with missing-file behavior
[ ] Close AMiOffice on the workstation(s) you will update (avoid locked EXE)

------------------------------------------------------------
2) SET VERSION (manual)
Open AMiOffice.vbproj and update:

Patch release:
[ ] Change <Version> X.Y.Z -> X.Y.(Z+1)
[ ] Do NOT change <AssemblyVersion>

Minor release:
[ ] Change <Version> X.Y.Z -> X.(Y+1).0
[ ] Do NOT change <AssemblyVersion>

Major release:
[ ] Change <Version> X.Y.Z -> (X+1).0.0
[ ] Change <AssemblyVersion> old -> (X+1).0.0.0

Notes:
- Do not set Version to 4 parts; keep it X.Y.Z.
- AssemblyVersion should remain A.0.0.0 between majors.

------------------------------------------------------------
